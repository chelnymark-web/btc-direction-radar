# 项目方案:BTC 方向罗盘 + 自动报告系统(交给 Claude Code 实施)

这份文档是给 Claude Code 的实施方案。请你(Claude Code)按"阶段"逐步实现,每个阶段做完**真的联网跑一遍验证**再进下一阶段——你运行在用户本地,没有出口白名单限制,Binance / Deribit / Yahoo 等都能直连,这正是要在本地做的原因。

## 一句话目标

一个**实时盯盘网页(方向罗盘)** + **日/周/月自动报告系统**:抓真实市场数据 → 按"拥挤度=方向、极端=警惕反转"的逻辑自动判方向 → 每项带中文解释 → 报告归档成可访问的网站。

## 设计哲学(必须贯穿全部实现,不只是 UI)

这套工具的判读逻辑不是"指标涨了就看多"。核心信条,写进代码注释和界面文案:
- **拥挤度即方向**:测的是多空哪一侧挤满了人,不是预测涨跌。
- **极端=警惕反转,不是顺势信号**:指标到极端(深色)读作"该侧已饱和,反向冲击最大",绝不是加仓依据。
- **近均衡=没有 edge**:多数指标在中性区时,市场近似有效,空手等待优于下注。
- 所有产出都附"这个数现在意味着什么"的一句话解释。
- 所有界面/报告结尾标注:个人研究工具,不构成投资建议。

## 技术栈

- 后端取数 / 报告:Python 3.11,只用 `requests` + 标准库 + `feedparser`,**不要上 pandas**。
- 前端:单文件 HTML + 原生 JS(无框架、无构建步骤),用 CSS 变量做深色主题。
- 托管 / 定时:GitHub Actions(cron) + GitHub Pages(`/docs` 目录)。
- 风格:深色"驾驶舱"仪表盘,等宽字体显示数据,横向罗盘指针表示方向。避免 AI 默认的三种套路(暖米色+衬线、纯黑+荧光绿、报纸风)。

## 数据源与可得性(诚实分层,别假装全有)

**A. 浏览器/Python 都能直拉(免费,核心仪表):**
- Binance 合约公开端点:资金费率(`/fapi/v1/premiumIndex`、`/fapi/v1/fundingRate`)、OI(`/futures/data/openInterestHist`)、大户多空比(`/futures/data/topLongShortPositionRatio`)、全体多空比(`/futures/data/globalLongShortAccountRatio`)、主动买卖(`/futures/data/takerlongshortRatio`)、季度基差(`/futures/data/basis`)、24h ticker、K线。
- Deribit 公开端点:指数价、DVOL(`/public/get_volatility_index_data`)、期权链(`/public/get_book_summary_by_currency`,本地用 Black-Scholes 反推 delta 算 25Δ skew 与期限结构)。

**B. 需要付费 key 或额外处理(用户本地可补,网页版做外链):**
- 清算热图 / 全网清算:Coinglass(API 收费;免费可外链或用 Binance `!forceOrder` websocket 自攒)。
- 链上(交易所流入流出、MVRV、已实现价、长短持有者成本):Glassnode / CryptoQuant(免费层薄;若用户有 key 就接,没有就外链)。
- CME 缺口:Yahoo Finance `BTC=F`(免费,本地可直拉,见阶段三)。
- 法案进度:Congress.gov(免费)+ 政策 RSS。

**对 A 类:网页直接拉;对 B 类:有 key 则在 Python 端接入并写进快照,无 key 则在网页做跳转入口。** 启动时先问用户有没有 Coinglass / Glassnode / CryptoQuant 的 key。

## 阶段一:实时方向罗盘网页(先交付这个,最快见效)

实现 `docs/dashboard.html`,单文件:
1. 顶部:实时价格 + 24h 涨跌 + 数据源状态(实时/快照/无数据)+ 手动刷新,每 60s 自动刷新。
2. **方向罗盘**:把下列指标按 `方向 × 极端等级` 加权,聚合成 −100~+100 读数,横向指针 + 一句话解读(极端时提示"该侧饱和、警惕反转";平衡时提示"近均衡、无 edge")。
3. 指标卡片(每张:数值、方向判定染色、一句话解释):资金费率、OI 24h 变化、大户多空比、全体多空比、主动买卖比、永续基差、季度基差、DVOL、25Δ skew(近月)、期限结构斜率。
4. 阈值与方向定义(可调,放在 JS 顶部常量):skew 正=怕跌=空头侧;费率/基差/多空比 正=多头侧;DVOL 不计方向只标恐惧水位。
5. 回退:浏览器直拉被 CORS/地区拦截时,自动读同目录 `./data/latest.json`(阶段二的 Actions 产物),状态标"快照"。

> 注:本目录 `docs/direction-compass.html` 就是一版可用实现(深色驾驶舱风、10 指标、BS 反推 skew、60s 刷新、回退快照)。你可以在它基础上改进:抽出阈值为可配置、加 30 日历史百分位判定(见阶段二)、补单元测试。**验证**:本地用浏览器打开,确认能直拉到实时数据并算出罗盘读数;断网测试回退逻辑。

## 阶段二:Python 取数 + 自动报告 + Actions

1. `scripts/fetch_data.py`:抓 A 类全部指标 + 算 skew/期限结构 + 拥挤度标记,输出 `data/latest.json` 与累积 `data/snapshots.jsonl`。**拥挤度判定:前 30 条用静态阈值,满 30 条后切换为"相对自身历史的百分位"判定**(更扎实)。
2. `scripts/fetch_news.py`:双模式——`daily` 抓过去 24h 新闻 RSS(CoinDesk/Cointelegraph/The Block/美联储);`weekly`/`monthly` 改为**未来事件前瞻日历**(见阶段四),输出 `data/news.json`。
3. `scripts/generate_report.py`:读快照 + 新闻/日历,渲染 `docs/reports/{kind}-{date}.html`,重建 `docs/index.html` 索引,复制 latest.json 到 `docs/data/` 供网页回退。**可选**:若设置 `ANTHROPIC_API_KEY`,调用 Claude API(model 用 `claude-sonnet-4-6`)生成一段中文简评(数据读数 + 新闻脉络/前瞻 + 巡检提示);没 key 则跳过,报告照常生成。
4. 三个 workflow(`.github/workflows/daily.yml` / `weekly.yml` / `monthly.yml`):cron 分别为每日、每周一、每月初(UTC,注意 +9 换算东京时间),pipeline = fetch_data → fetch_news → generate_report → commit。`permissions: contents: write`。
5. 单项失败要 `try/except` 优雅降级,不能让整个 pipeline 崩。

> 本目录 `scripts/` 下就是这套脚本的可用实现,可直接复用并改进:把网页的方向罗盘逻辑与 Python 端的拥挤度判定对齐(同一套阈值/百分位)。

## 阶段三:CME 缺口模块(用户明确要的,需真 CME 数据)

`scripts/fetch_cme_gaps.py`:
- 数据源:Yahoo Finance `BTC=F`(CME 连续合约日线),query1 失败退 query2,再退 Stooq `btc.f`,最后兜底 Binance 现货近似(标注"近似")。需带浏览器 UA。
- 算法:遍历每个周末,周五 close(C)与下一交易日 open(O),`C!=O` 记缺口 `[min,max]`;用其后所有日线的 low~high 判定是否触及(触及=已回补,记 partial)；输出落在 [10000,130000] 且未回补的全部缺口,附距现价% 与上/下方。
- 输出 `data/cme_gaps.json`,接进报告(表格)与网页(画成横线叠在价格刻度上,最近几个高亮)。
- **验证**:真的从 Yahoo 拉到 2017 至今数据,打印未回补缺口总数与离现价最近 5 个,人工对照 TradingView `CME:BTC1!`。

## 阶段五(可选增强):消息冲击模拟器

## 阶段六(已起头):MSTR 杠杆飞轮监控

已加入文件:`data/mstr.json`(手动维护的慢变量+阈值)、`scripts/fetch_mstr.py`(用实时 BTC 价+慢变量算 mNAV/距成本/股息覆盖,输出 `data/mstr_status.json`)、罗盘网页 `docs/direction-compass.html` 已含 MSTR 监控区。

**诚实约束**:MSTR 的 mNAV/持仓/卖币/股息覆盖**没有干净的免费实时 API**。所以:
- 慢变量(持仓、平均成本、债务、股息、股数)从 `data/mstr.json` 手动维护——看到新的 SEC 8-K(每周一前后的持仓公告)或季报就更新数字。**注意**:聚合站(bitcointreasuries 等)的 avg_cost 可能是过时缓存(曾显示 66,384 旧值),以 SEC 8-K 披露为准(截至 2026-05-18 为 75,700)。
- mNAV 要真实,需 MSTR 市值=股价×流通股;若你有股票行情源,把市值填进 `last_mstr_marketcap_musd`,脚本会用真实值算;否则用 `premium_hint` 仅作提示。
- Claude Code 可增强:接 SEC EDGAR(cik 0001050446)自动解析最新 8-K 的持仓/成本;接一个股票行情源自动算 mNAV;把"卖币规模"做成时间序列(规模扩大=Saylor 铁律破裂的行为信号)。

**监控阈值与含义**:mNAV<1.0=飞轮反转(增发变折价稀释,被迫卖币概率跳升);BTC<成本线 75,700=仓位浮亏(现价已在此线下约 17%);股息覆盖<9 月=融资压力;已开始卖币(2026-06-01)规模扩大=铁律实质破裂。这几个是"先于 Saylor 公开表态泄露真相"的硬指标——读 8-K,别听 X。

## 阶段七(已起头):关键位 + 斐波那契 + 重合(confluence)

已加入:`data/levels.json`(手动维护的支撑/阻力/均线/缺口/MSTR成本 + 斐波那契摆动区间)、`scripts/fetch_levels.py`(算斐波那契回撤/扩展位,与所有位合并找重合带,输出 `data/levels_status.json`)、罗盘网页已含"关键位重合带"区。

**核心理念(必须保留)**:斐波那契和单条支撑/阻力**单独看是噪音**——它的有效性只来自"协调焦点"(够多人盯着同一条线下单)。所以本模块**只高亮多来源重合的带**(斐波那契+均线+缺口+MSTR成本叠在一起处),单线弱化处理。重合带越多来源、越靠近现价,越值得重视。

**维护**:`fib_swing` 的 high/low 取你认定的当轮趋势端点(变盘了就改);`levels` 里的支撑阻力/均线随行情更新;`cme_gaps` 的确切边界用本地 TradingView `CME:BTC1!` + CME Gap 指标确认后填入。`confluence_pct`(默认 1.5)调大会让更多位并成带。

**Claude Code 可增强**:均线(50/200 日)可由 K 线自动算而非手填;CME 缺口可接阶段三的 `fetch_cme_gaps.py` 自动喂入;斐波那契摆动区间可由近 N 日高低点自动取。

## 阶段九(最高优先补全):全球股市 / 宏观关联维度

**这是当前 BTC 最高关联、却还没接进指挥台的维度——必须补。** 因为 2026 年 BTC = 高 beta 科技股(与纳指相关 +0.7~0.8),BTC 的反弹天花板是股市定的,股市回调时 BTC 放大 1.5-2 倍地跌。指挥台缺了它,概率判断就缺了最重要的一块。

在 `command-deck.html` 加一个"宏观"维度区(并给它**高权重**,建议提到与衍生品相近的 35-40%,相应下调其它),包含:
1. **纳指期货 / QQQ 方向**(最高关联,实时领先 BTC):涨=偏多分,跌=偏空分。数据源:Yahoo Finance `^NDX` 或 `QQQ`(本地可拉,容器被白名单挡)。
2. **DXY 美元指数**(看 100 关口):>100 且上行=流动性紧=偏空;数据源 Yahoo `DX-Y.NYB` 或 `DX=F`。
3. **VIX**(恐慌水位):>25=risk-off 转向信号;Yahoo `^VIX`。
4. **油价**(布伦特/WTI,通胀火星):急升=强化鹰派+滞胀=偏空;Yahoo `BZ=F`/`CL=F`。
5. **(可选)标普广度**:200 日均线上方成分股比例,广度崩坏=顶部信号。数据源较难免费拉,可先留外链。
6. **(可选)黄金**:仅在与 BTC 罕见同向下跌时高亮=恐慌极端/去货币化信号;Yahoo `GC=F`。

**方向合成逻辑**:纳指/股市偏弱 + DXY 强 + VIX 升 + 油价升 = 宏观偏空分(对 BTC 利空)。把这个宏观分作为新维度并入总概率。

**重要**:Yahoo Finance 接口浏览器直拉常被 CORS 拦,Claude Code 在本地可用 yfinance(Python)在 Actions 端取数写入快照,网页回退读快照——和现有 latest.json 回退机制一致。优先走"Actions 端 Python 取 + 网页回退"这条路,比浏览器直拉稳。
